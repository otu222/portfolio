USING OPENSTREETMAP DATA TO CREATE D&D MAPS

This map was created in QGIS v3.32.2 This map utilized US state and county data from Natural Earth Data for filtering what data would be imported through other tools. OpenStreetMap Data was collected through the QuickOSM v2.2.3 plugin for QGIS. Of note, These keys below were used within the QuickOSM tool to gather the data represented in these maps: amenity building highway shop These keys were set to a Layer extent of Fayette County in Lexington, KY That layer was produced by filtering the Natural Earth US County shapefile to just Fayette County. The projected coordinate system used was EPSG:3089 [NAD83 / Kentucky Single Zone (ftUS)] The scaling for each section of the map varied and is listed underneath each section From Left to Right: (1:500), (1:2500), (1:50000)

Turtle Back Arch and 718 named arches

Arch locations are extracted from the Domestic and Antarctic Names (State and Topical Gazetteer) published by the United States Board on Geographic Names. (geonames.usgs.gov). Data accessed August, 2018. Elevation contours shown have an interval of 20 feet with labeled index contours every 100 feet. Vector data was extracted from a USGS NED 1/3 arc-second raster digital elevation model (approximately 10-meter resolution). This data can be accessed at the USGS 3D Elevation Program (usgs.gov).
